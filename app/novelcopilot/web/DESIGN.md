# 디자인 시스템 — 웹소설 코파일럿

> 컨셉: **Modern Writing Studio** — 조용하고 단단한, 순백 캔버스 위의 집필 도구.
> 화면은 "기능 데모"가 아니라 "작가가 매일 쓰는 운영 서비스"처럼 보여야 한다.
> 화려함이 아니라 **여백·활자·위계**로 완성도를 만든다.

---

## 1. 원칙

1. **순백 캔버스.** 순백(#FFFFFF) 표면 위 중성 잉크 텍스트. 따뜻한 종이톤은 쓰지 않는다(모던·선명). 면 분리는 쿨 그레이(`--surface-2`)와 1px 헤어라인으로.
2. **본문만 명조.** 소설 본문 리더(회차·뷰어)만 명조 세리프 — 진짜 책처럼 읽힌다. 그 외 UI 크롬(브랜드 워드마크·제목·라벨·버튼)은 **제목까지 전부** Pretendard 산세리프로 통일해 모던 톤을 지킨다.
3. **색은 의미일 때만.** 액센트(에디토리얼 블루)는 링크·포커스·선택에만. 상태색(초록/호박/빨강)은 의미를 가질 때만. 그 외엔 무채색.
4. **하드웨어가 아니라 작품.** 내부 기술 용어를 화면에 노출하지 않는다(§5 용어 사전). 작가는 "온톨로지"가 아니라 "공식 설정"을 본다.
5. **여백 > 선 > 그림자.** 경계는 우선 여백으로, 다음 헤어라인, 그림자는 최소.

---

## 2. 색 (CSS 변수)

```
--paper:      #FFFFFF   /* 앱 배경 — 순백 */
--surface:    #FFFFFF   /* 카드·패널 */
--surface-2:  #F6F7F9   /* 미묘한 채움·hover·로그/코드 — 쿨 그레이 */
--ink:        #111317   /* 본문·1차 텍스트·1차 버튼 — 중성 잉크 */
--ink-soft:   #3A3F46   /* 부제 */
--muted:      #6B7280   /* 보조 텍스트 (순백 대비 ~4.8:1, AA 충족) */
--muted-soft: #566070   /* 색틴트(ok/warn-soft) 배경 위 보조문 — AA */
--line:       #ECEEF1   /* 헤어라인 보더 — 중성 */
--line-strong:#DCDFE4   /* 강조 보더 */
--brand:      #1E2A4A   /* 딥 잉크네이비 — 시그니처: 워드마크·히어로 강조·표지·강조면 좌보더 */
--brand-soft: #EEF1F8
--accent:     #2563EB   /* 링크·포커스·선택 (모던 블루). tint: --accent-100/-700 */
--accent-soft:#EFF4FF
--ok:#0F7A4B  --ok-soft:#E6F4EC     /* 완성·정상 (soft 위 AA 4.5:1) */
--warn:#8F5D0F --warn-soft:#FBF0DC  /* 검토 필요·주의 (soft 위 AA) */
--bad:#C4231F --bad-soft:#FCEAE9    /* 충돌·실패 (soft 위 AA) */
```

- **1차 버튼은 잉크 블랙**(순백 글자) — 그라데이션 금지. 보라색 그라데이션은 절대 쓰지 않는다(AI 클리셰).
- 그림자: `--shadow: 0 1px 2px rgba(16,17,23,.05), 0 6px 20px rgba(16,17,23,.06)` — 중성 검정 알파, 아주 옅게.

## 3. 활자

```
--font-sans:  "Pretendard", -apple-system, "Segoe UI", "Malgun Gothic", sans-serif
--font-serif: "Noto Serif KR", "Nanum Myeongjo", Georgia, serif
```

- **세리프(명조)**: **소설 본문 리더(회차·뷰어)에만**. → 책/원고의 읽기 정체성. 브랜드·제목엔 쓰지 않는다.
- **산세리프(Pretendard)**: 브랜드 워드마크·히어로/작품 제목 포함, 그 외 모든 UI(버튼·탭·라벨·인스펙터).
- 위계: **디스플레이 2.5rem/800(.display·히어로)** / 섹션 1.12rem/700 / 카드제목 1rem/650 / 본문 15px / 캡션 0.8rem. 자간 -0.03em(디스플레이)·-0.01~-0.02em(제목). 웨이트 폭 800/700/650/600/500/400 으로 같은 크기라도 굵기로 위계. 행간 본문 1.7 / 리더 1.95.
- **시그니처 정체성**: 종이 개성을 버린 자리는 (1) 딥 잉크네이비 `--brand` 절제 사용(워드마크·히어로 eyebrow·강조면 좌보더), (2) 디스플레이 타이포 1단으로 세운다. 색을 늘리지 않고 '이 제품의 색·활자'를 고정 — 보라 그라데이션 AI클리셰는 계속 금지.

## 4. 컴포넌트 규약

- **radius**: 카드 14px · 컨트롤 9px · 칩 999px.
- **간격 스케일**: 4 / 8 / 12 / 16 / 24 / 32 px.
- **버튼**: 1차=잉크 채움, 2차=헤어라인 아웃라인(투명 배경), 위험=빨강 텍스트. hover는 명도 1단계.
- **포커스**: `box-shadow: 0 0 0 3px var(--accent-soft)` + 보더 액센트. 아웃라인 제거 금지(접근성) — 링은 항상 보이게.
- **상태 배지**: 채움 약한 pill(상태색 soft 배경 + 상태색 텍스트). "완성"=초록 / "검토 필요"=호박 / "충돌"=빨강.
- **칩**: 적용=초록 soft, 보류=빨강 soft, 중립=무채색.
- **빈 상태**: 항상 한 문장 안내 + 다음 행동 유도(뮤트 텍스트).

## 5. 용어 사전 (내부 → 화면)

화면에는 **오른쪽** 표현만 쓴다. 내부 코드·주석·문서는 왼쪽 유지.

| 내부(기술) | 화면(사용자) |
|---|---|
| 온톨로지 / SSOT / ground_truth / 박기 | **공식 설정** (작가가 확정한 사실) |
| LLM Wiki / compounding wiki / watermark | **작품 노트** (자동 정리) |
| lint | **자동 점검** |
| 에이전틱 하네스 | **회차 집필** / 진행 상황 |
| worldgen / 세계 생성 | **작품 세계 만들기** |
| FINALIZED | **완성** |
| ESCALATED / escalation | **검토 필요** |
| promote / 캐논으로 박기 | **공식 설정으로 확정** |
| 캐논(badge) | **공식** |
| spine / 평면 모드 | **이야기 구조** |
| arc / episode | **단락(아크)** / **에피소드** |
| climax | **절정** |
| drift / 드리프트 | **전개 점검** |
| provisional / 동적추가 | **AI 추가 (미확정)** |
| narrative_inferred / 추정 관계 | **추정 (미확정)** |
| lazy 생성 | **진행하며 자동 설계** |
| 콜 / 토큰 | **AI 사용량** |
| 빨간 펜 | **작가 지시** |

원칙: 명사는 "작가가 출판사에서 들을 법한 말"로. 영문 상태값(FINALIZED 등)·내부 자료구조명(spine, watermark)은 화면 노출 금지.

## 6. 적용 파일

- `style.css` — 위 토큰·컴포넌트 전부.
- `index.html` — 폰트 로드, 라벨/문구 용어 적용.
- `app.js` — 동적 문구·배지·`NODE_LABELS`(진행 로그 한글화). 그래프(cytoscape) 색은 `cssVar()`로 CSS 토큰에서 런타임 조회 — 색 하드코딩 금지(종이톤 회귀 차단).
