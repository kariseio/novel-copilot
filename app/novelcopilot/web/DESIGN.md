# 디자인 시스템 — 웹소설 코파일럿

> 컨셉: **Editorial Writing Studio** — 조용하고 단단한, 종이와 잉크의 집필 도구.
> 화면은 "기능 데모"가 아니라 "작가가 매일 쓰는 운영 서비스"처럼 보여야 한다.
> 화려함이 아니라 **여백·활자·위계**로 완성도를 만든다.

---

## 1. 원칙

1. **종이와 잉크.** 따뜻한 종이톤 배경 위 잉크 블랙 텍스트. 순백·순흑은 쓰지 않는다(눈의 피로·차가움).
2. **활자가 주인공.** 소설 본문은 명조 세리프로 — 진짜 책처럼 읽힌다. UI 크롬은 Pretendard 산세리프.
3. **색은 의미일 때만.** 액센트(에디토리얼 블루)는 링크·포커스·선택에만. 상태색(초록/호박/빨강)은 의미를 가질 때만. 그 외엔 무채색.
4. **하드웨어가 아니라 작품.** 내부 기술 용어를 화면에 노출하지 않는다(§5 용어 사전). 작가는 "온톨로지"가 아니라 "공식 설정"을 본다.
5. **여백 > 선 > 그림자.** 경계는 우선 여백으로, 다음 헤어라인, 그림자는 최소.

---

## 2. 색 (CSS 변수)

```
--paper:      #FAF8F3   /* 앱 배경 — 따뜻한 종이 */
--surface:    #FFFFFF   /* 카드·패널 */
--surface-2:  #F4F2EC   /* 미묘한 채움·hover */
--ink:        #1C1B19   /* 본문·1차 텍스트·1차 버튼 */
--ink-soft:   #3D3A34   /* 부제 */
--muted:      #8C877B   /* 보조 텍스트 */
--line:       #E7E3D9   /* 헤어라인 보더 */
--line-strong:#D8D3C6   /* 강조 보더 */
--accent:     #2F5E8C   /* 링크·포커스·선택 (에디토리얼 블루) */
--accent-soft:#EAF1F8
--ok:#3C7D5A  --ok-soft:#E7F2EB     /* 완성·정상 */
--warn:#B5781F --warn-soft:#FBF1DE  /* 검토 필요·주의 */
--bad:#B23A38 --bad-soft:#F7E9E7    /* 충돌·실패 */
```

- **1차 버튼은 잉크 블랙**(종이색 글자) — 그라데이션 금지. 보라색 그라데이션은 절대 쓰지 않는다(AI 클리셰).
- 그림자: `--shadow: 0 1px 2px rgba(28,27,25,.04), 0 6px 20px rgba(28,27,25,.05)` — 따뜻한 잉크 틴트, 아주 옅게.

## 3. 활자

```
--font-sans:  "Pretendard", -apple-system, "Segoe UI", "Malgun Gothic", sans-serif
--font-serif: "Noto Serif KR", "Nanum Myeongjo", Georgia, serif
```

- **세리프(명조)**: 브랜드 워드마크 · 히어로 제목 · **소설 본문 리더**. → 책/원고의 정체성.
- **산세리프(Pretendard)**: 그 외 모든 UI(버튼·탭·라벨·인스펙터).
- 위계: 히어로 1.7rem / 섹션 1.05rem / 본문 0.93rem / 캡션 0.8rem. 자간 -0.01em(제목), 행간 본문 1.7 / 리더 1.95.

## 4. 컴포넌트 규약

- **radius**: 카드 14px · 컨트롤 9px · 칩 999px.
- **간격 스케일**: 4 / 8 / 12 / 16 / 24 / 32 px.
- **버튼**: 1차=잉크 채움, 2차=헤어라인 아웃라인(투명 배경), 위험=빨강 텍스트. hover는 명도 1단계.
- **포커스**: `box-shadow: 0 0 0 3px var(--accent-soft)` + 보더 액센트. 아웃라인 제거 금지(접근성) — 링은 항상 보이게.
- **상태 배지**: 채움 약한 pill(상태색 soft 배경 + 상태색 텍스트). "완성"=초록 / "검토 필요"=호박 / "충돌"=빨강.
- **칩**: 적용=초록 soft, 보류=빨강 soft, 중립=무채색.
- **빈 상태**: 항상 한 문장 안내 + 다음 행동 유도(뮤트 텍스트).

## 5. 용어 사전 (내부 → 화면)

화면에는 **오른쪽** 표현만 쓴다. 내부 코드·주석·문서는 왼쪽 유지.

| 내부(기술) | 화면(사용자) |
|---|---|
| 온톨로지 / SSOT / ground_truth / 박기 | **공식 설정** (작가가 확정한 사실) |
| LLM Wiki / compounding wiki / watermark | **작품 노트** (자동 정리) |
| lint | **자동 점검** |
| 에이전틱 하네스 | **회차 집필** / 진행 상황 |
| worldgen / 세계 생성 | **작품 세계 만들기** |
| FINALIZED | **완성** |
| ESCALATED / escalation | **검토 필요** |
| promote / 캐논으로 박기 | **공식 설정으로 확정** |
| 캐논(badge) | **공식** |
| spine / 평면 모드 | **이야기 구조** |
| arc / episode | **단락(아크)** / **에피소드** |
| climax | **절정** |
| drift / 드리프트 | **전개 점검** |
| provisional / 동적추가 | **AI 추가 (미확정)** |
| narrative_inferred / 추정 관계 | **추정 (미확정)** |
| lazy 생성 | **진행하며 자동 설계** |
| 콜 / 토큰 | **AI 사용량** |
| 빨간 펜 | **작가 지시** |

원칙: 명사는 "작가가 출판사에서 들을 법한 말"로. 영문 상태값(FINALIZED 등)·내부 자료구조명(spine, watermark)은 화면 노출 금지.

## 6. 적용 파일

- `style.css` — 위 토큰·컴포넌트 전부.
- `index.html` — 폰트 로드, 라벨/문구 용어 적용.
- `app.js` — 동적 문구·배지·그래프 라이트테마 색·`NODE_LABELS`(진행 로그 한글화).
