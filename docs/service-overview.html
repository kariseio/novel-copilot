기계가 쓰고,
작가가 조향한다.

시드 한 줄 → 세계관·설정집·서사 골격 생성 → 일관성 엔진이 검증하는 회차 단위 집필.
수백 화를 써도 설정이 무너지지 않게 — 기억은 기계가, 결정은 작가가.

실선 = 캐논(박기) · 위반 시 차단/재작성 점선 = 잠정/참고(찾기) · 비구속 빨간 펜 = 작가 권한 · 캐논을 바꾸는 유일한 손
01

핵심 원리 — 비대칭 일관성

이 한 장이 시스템 전부의 뼈대

LLM 장편 생성이 무너지는 1순위 원인은 기억 오염 — AI가 스스로 추론한 사실이 설정에 섞여 들어가 오류가 복리로 쌓이는 것. 이 시스템은 기억을 두 신뢰 등급으로 물리적으로 분리해 오염을 구조적으로 차단한다.

박기 — ground_truth binding

  • 인물 속성 (눈색·등급·신분 …)
  • 생애주기 상태 (사망·각성·발각 — 비가역 선언 가능)
  • 관계 엣지 (작가가 직접 그은 것)
  • 세계 시간선 (몇 화부터 유효한가)
프롬프트에 무조건 주입되고, 위반하면 자동 재작성/차단된다. 여기 들어오는 길은 작가의 명시 행위뿐.

찾기 — narrative non-binding

  • 회차 본문 RAG 검색 · LLM 위키
  • 누적 줄거리 요약 (계층 롤업)
  • 설정집 산문 다이제스트 · 아크 앵커
  • AI 자동추출 사실 · 월드젠 제안 (잠정)
맥락으로 참조만 된다. 아무리 그럴듯해도 스스로 캐논이 되지 못한다 — 자동 승격 금지가 코드로 강제됨.
AI의 모든 산출물은 점선으로 태어나고, 작가의 빨간 펜만이 실선으로 만든다.

02

회차가 만들어지는 길

제어 흐름은 전부 코드 · LLM은 쓰기에만

세계 생성 자동

시드(장르·전제 한 줄)로부터 — 장르에 맞는 추적 축만 설계(레벨·사망 강제 없음).

추적 속성·생애주기인물 + 보이스 카드엔딩-주도 아크/에피소드 골격장르 설정집(13+ 카테고리)집필 스타일·끝맺음 정책

비트 설계 자동

현재 에피소드의 절정(그 장르의 카타르시스)으로 수렴하는 이번 화 설계. 작가 지시는 권위 슬롯에, 미회수 복선은 참고 슬롯에 분리 주입.

집필 루프 자동 게이트

장면 초안 → 일관성 검사 → 위반 시 해당 부분만 재작성(최선본 보존) → 수렴 실패 시 작가 에스컬레이션.

사망·제거 상태 행동 차단속성 불일치등급 역행관계 상충(동맹↔적대)증거 스팬 없는 클레임 폐기

기억 갱신 자동

확정 회차에서 — 신규 인물·관계·상태를 추출해 점선(잠정)으로 적재. 요약·RAG·위키 갱신.

작가 개입 빨간 펜

언제든 — 지시(이번 화부터 누적 적용), 잠정 제안 승격/기각, 관계 수정, 설정집 편집, 문체·정책 변경.

완결까지 자동 항해: 엔딩을 먼저 고정하고 역순으로 아크를 설계 — 에피소드가 소진되면 다음 아크를 이어 생성하고, 모든 아크가 끝나면 마지막 화는 절단 없이 여운으로 닫고 완결 처리된다.
03

기능 지도

작가가 화면에서 만나는 것

회차 생성·열람

  • SSE 실시간 하네스 가시화(검사·재작성 과정) 자동
  • 빨간 펜 지시 — 이번 화부터 누적 전파 입력
  • 에스컬레이션(수렴 실패) 처리 제안→승인

온톨로지 그래프

  • 실선(캐논)/점선(AI 추정) 시각 구분 자동
  • 관계 긋기·종료 — 자유 타입+상태+관점(pov) 입력
  • 시간선 모순(사망 후 관계 등) 사전 차단 게이트

스토리 바이블

  • 장르 템플릿 산문 설정(마법·세력·감정선…) 자동
  • 편집·추가·삭제 입력
  • "캐논으로 박기" — 세계규칙 승격(추적·주입) 입력

서사 구조

  • 엔딩·아크·에피소드·진행 커서 열람 자동
  • 드리프트 경보(필수인물 미등장·분량 초과) 경보
  • 복선 적체 경보 — 회수는 작가 선택 경보

대화형 세계 구축

  • 대화로 인물·관계·설정 제안 → 잠정 착지 제안
  • 그래프·설정집 실시간 반영(점선) 자동
  • 되묻는 질문으로 세계 심화 제안

엔진 내부 관찰

  • 온톨로지 SSOT(속성·시간선·신뢰등급) 자동
  • LLM 위키 + 결정론 lint(미회수 복선 등) 자동
  • 비용 계측(단계별 토큰) 자동

누가 결정하는가 — 3모드 계약

모든 기능은 셋 중 하나
규칙은 하나다 — 캐논을 바꾸는 행위는 전부 작가 입력. 시스템 자동은 비구속(점선)까지만 쓴다. 그 사이, 시스템이 제안하고 작가가 확정하는 것이 인터랙션이다.
모드해당 기능경계
① 자동 RAG 색인·검색 / 누적 요약·계층 롤업 / 위키 / 회차 자동추출 / 일관성 게이트·부분 재작성 / 드리프트·복선 계산 / 비용 계측 / 보이스 카드 주입 캐논 불변. 산출물은 점선(잠정)으로만 착지. 예외 1건: 가역 상태의 전진(소속 변경 등)은 모순 검사 후 자동 기록
② 인터랙션 월드젠 제안 → 승격 대기 / AI 추정 관계(점선) → 작가 확정 / 에스컬레이션 처리 / 드리프트·복선 적체 경보 대응 시스템은 제안·경보까지. 확정 행위 자체는 ③으로 넘어감
③ 작가 입력 캐논 승격("박기") / 관계 긋기·종료 / 인물 추가 / 빨간 펜 지시 / 설정집 편집 / 문체·정책(끝맺음·복선 리마인더·persona·분량) / 회귀 허용 스위치 캐논과 스티어링의 유일한 변경 경로

정직한 경계

알고 있는 미해결
재미는 별도 문제다. 이 엔진은 "설정이 안 어긋남"을 보장한다 — 재미(페이싱·사이다)는 보이스 카드·복선 되먹임·프록시 측정으로 접근 중이며, 실독자 블라인드 평가로 검증돼야 한다.
이야기-시간 단일 축. 사건 시각과 서술 순서가 한 좌표 — 미스터리 알리바이·회귀 전생 타임라인의 정밀 분리는 다음 구조 과제.
단일 사용자 운영. 단일 프로세스 계약(동시 작가·수평 확장은 영속 이관 후).
사람이 필요한 검증. 실작가 인터뷰 · 실독자 선호 평가 · 플랫폼 AI 정책 조사 — 도구와 프로토콜은 준비됨, 실행 대기.
