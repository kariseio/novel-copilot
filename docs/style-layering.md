# 설계: 문체 레이어링 — 보편 바닥 + 작가 문체 오버레이

> 출발 통찰: **"문체는 작가마다 다르다"**(헤밍웨이 균일 단문 ↔ 포크너 만연체, 둘 다 사람·둘 다 거장). 따라서 보편 "잘 쓴 문체" 기준은 없고, 기본 문체 규칙을 *하나의 미학*으로 강요하면 다른 문체를 쓰려는 작가와 충돌한다.
>
> 관련: [ai-style-analysis.md](ai-style-analysis.md)(AI티 측정 — 같은 통찰의 측정 측면).

---

## 1. 문제: 기본 규칙이 '안 AI스럽게'라는 명목으로 한 미학을 강요했다

`DEFAULT_STYLE_RULES` 8개는 성격이 다른 3가지를 한 바구니에 섞고 있었다:

| 종류 | 정체 | 작가-불변? | 예(기존 규칙) |
|---|---|---|---|
| **① 모델 아티팩트** | 번역투·이중피동·기계적 병렬·헤징·풍선직유 | 불변(누구의 스타일도 아닌 모델 잔재) | 규칙4 "번역투 대신", 규칙5 "삼단 병렬 드물게" |
| **② 장르·매체 장인성** | 모바일 조판·대사 호흡·절단신공·분량 | 거의 불변(시장 제약) | 규칙1 조판, 규칙6 절단, 규칙8 분량/시제 |
| **③ 작가 보이스** | 문장 리듬·감정 처리·직유 밀도·서술 거리·어휘 격 | **가변** | 규칙2 "보여주기", 규칙5 "문장길이 들쭉날쭉" |

핵심 결함: **규칙 2·5가 ③(작가 스타일)을 보편 바닥에 박아놨다.** "감정을 라벨로 말하지 말라"·"문장 길이를 변주하라"는 *하나의 미학*이지 AI 티가 아니다 — 직접 서술하는 전지적 무협도, 의도적으로 균일한 단문(헤밍웨이)도 사람 글이다.

## 2. 3층 모델

```
Layer 0  하드 바닥 (override 불가)   = ① 아티팩트 억제 + ② 시장/연속성 제약(분량·모바일 가독·시점/시제 일관·호칭)
Layer G  장르 (GenreContract)        = 어휘 격·쾌감 엔진 (기존)
Layer 1  소프트 디폴트 (override 가능) = 기존 8규칙의 의견 있는 미학(보여주기·문장 변주·대사주도·사이다 확정서술)
Layer 2  작가 오버레이               = 작품별 서술 문체 — Layer 1의 미학 축을 작가 지정이 덮어씀
```

기본 8규칙은 **소프트 디폴트로 유지**(신규 작가에게 멀쩡한 보이스 제공, 빈 화면 회피). 자기 문체가 뚜렷한 작가는 오버레이로 규칙 2·5 같은 축을 갈아끼운다. **Layer 0(아티팩트+시장)만 누구도 못 끈다** — 번역투를 원하는 작가는 없고, 5,000자는 플랫폼 제약.

## 3. 이번에 출하한 것 (구조적 최소 슬라이스)

| 파일 | 변경 |
|---|---|
| `domain/world.py` | `StyleSpec.author_style: str = ""` — Layer 2 슬롯(빈 기본값) |
| `engine/prompts.py::render_style` | author_style 설정 시 "[작가 지정 문체 — 미학 축 우선]" 블록 append + precedence 선언. **빈 값이면 추가 0(무회귀, 검증)** |
| `api/schemas.py` | `StylePolicyRequest.author_style` |
| `services/copilot.py` | setStylePolicy 패치(strip + 2000자 cap). evict→다음 요청 재구성 |
| `engine/harness.py` | draft_ctx 디버그에 author_style 노출 |
| `web/app.js` | 생성정보 탭에 "작가 지정 문체" 표시(esc 이스케이프) |
| `web/{index.html,app.js,style.css}` | **작가 설정 UI(Iteration 2)** — 설정집 > **문체** 탭. author_style textarea(2000자 cap·aria-label) + 끝맺음 정책 select → `PUT /style`. 저장 후 전체 재조회(stale 혼합 방지), '비우기'는 2스텝 안내, XSS esc |

**precedence/충돌 규칙**: 미학 축(문장 리듬·감정 처리·직유 밀도·서술 거리·어휘 격)이 충돌하면 **작가 오버레이 승**. 바닥 제약(분량·시점/시제·모바일 가독·호칭)은 **항상 유지** — 단 분량·시제는 프롬프트 문장이 아니라 **결정론 게이트(`norm`·`_fix_tense`·`tense_leak_ratio`)가 실제 방어선**이다(프롬프트는 보조).

## 4. 측정(ai_tell)과의 정합

같은 seam으로 갈린다: Layer 0 아티팩트 → 작가-불변 신호(쉼표·번역투)로 측정(보편 정당). Layer 2 보이스 → "AI인가"가 아니라 **작가 프로필 대비 충실도/드리프트**로 측정. 즉 레이어링이 ai_tell을 자동으로 "검출기→충실도 계기"로 재정의한다.

## 5. 의도적으로 안 한 것 (투기적 — 검증 후)

- **작가 노브 UI**(단문↔만연/대사비중/서술거리 슬라이더) — 노브 택소노미·UX 검증 필요.
- **작가 노브 UI(슬라이더)** — 자유 텍스트 에디터는 출하(설정집>문체). 단문↔만연/대사비중/서술거리 *구조화 노브*는 택소노미·UX 검증 필요로 보류.
- **퇴고→문체 학습 루프** — 작가지시 퇴고를 문체 supervision으로 누적해 author_style 자동 추론(데이터 축적 필요).
- **예시 글 → 문체 서술자 추출** — raw 주입은 자기표절 진원(규칙 주석 경고)이라 추출/회전 필요.
- **8규칙의 하드/소프트 물리 분할** — 지금은 precedence 텍스트로 선언만. 규칙 문자열을 외과적으로 재작성하면 회귀 위험(갓 커밋된 규칙) → 보류.

## 6. 적대 리뷰가 남긴 미해결·주의 (code-critic)

- **precedence 자연어 지시 효능 — ✅ 실측 검증(2026-06)**. 충돌 오버레이("균일 단문·비유 금지" vs 규칙5 변주·규칙4 비유)로 A/B(2개 장면 — 액션·감정, baseline vs conflict 각 N=4·6, ai_tell로 측정):
  - **sent_len_cv**: conflict가 baseline 대비 일관 하락(감정 장면 0.464→**0.369**, ~20%; conflict 범위 0.31–0.43이 baseline 0.40–0.54 아래에 대부분 위치).
  - **simile_per_1k**: baseline 4/6 샘플에 직유(최대 3.88), conflict는 **5/6이 0**(1건만 누출).
  → **자연어 precedence가 방향성 있게·꽤 일관되게 작동.** 완벽한 하드 스위치는 아니나(직유 1건 누출·CV 범위 일부 겹침) 미학 축 조향엔 충분 → **구조화 입력 전환 불필요**. *완충: 분량·시제는 게이트가 강제.*
  - 한계: 단일 모델·강한 어조 오버레이·소표본·동일 축 측정. 결정적 단언엔 대표본·다모델 필요. (probe는 throwaple, 미커밋)
- **사망교정/rewrite 패스(harness:159)에도 미학 오버레이 주입(minor)** — 이 패스는 "문체 보존하며 캐논 위반만 교정"인데 미학 오버레이가 붙어 교정 폭을 키울 수 있음. floor-only 렌더 분리는 측정 후 결정.
- **`update_style` 엔드포인트 인증/소유권(기존 IDOR — 전 라우트 공통)** — Iteration 2 리뷰 재확인: `/style` 포함 **모든 라우트에 Depends 인증 0건**. author_style을 작가 정식 입력으로 UI 노출하며 프롬프트 인젝션 표면이 승격됨. **결정: 단일 작가 로컬 PoC라 현재 비악용, 인증은 멀티유저 전환 시 전 라우트 일괄 처리(이 UI 단독 책임 아님). UI 추가 자체는 이미 무인증이던 엔드포인트에 버튼만 더한 것이라 위험 델타 최소.**
- **author_style 회차별 중복 영속(draft_ctx 240자)** — B2 비대 패턴과 동근원. 참조화 검토.
