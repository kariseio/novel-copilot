# 분석: 한국어 웹소설의 'AI 티' — 정체·검출 한계·우리가 적용한 측정

> 5라운드 리서치(웹소설 문체 관행 · AI 검출 OSS/스킬 · KatFishNet 측정자질 · 검출기 신뢰도 · 커뮤니티 기법) 종합 + 코드 적용 기록.
>
> 목적: "AI스럽지 않게 쓴다"는 요구를 **규칙 열거(두더지잡기)가 아니라 측정-주도**로 코어에 안착시키는 근거와 그 구현(`engine/quality_gates.py::ai_tell_profile`)의 설계·검증·한계를 문서화한다.

## 0. 요약 (Abstract)

"AI 티"는 단어 사전으로 잡히는 표면 현상이 **아니다**. 실측·리서치가 수렴한 결론은 두 가지다.

1. **AI 티의 절반은 문체가 아니라 일관성 붕괴**다 — 설정·호칭·시제·인물 보이스가 회차를 가로질러 미끄러지는 것. 이건 프롬프트 문장으로 못 고치고 **소스(장면-스코프 주입·구조화 상태)에서** 막아야 한다.
2. **나머지 절반(문체 게슈탈트)은 규칙으로 못 고친다** — 패턴을 금지하면 풍선효과로 다른 패턴이 부푼다(조판을 누르면 직유가 튀어나온다). 실측에서 반복 확인됐다.

그래서 우리가 한 일은 **금지 규칙 추가가 아니라 측정 신호 추가**다: KatFishNet(ACL 2025)이 정의한 한국어 AI 티 분포자질을, 외부 코드 import 없이·LLM 0콜·결정론으로 재구현한 `ai_tell_profile()`. **판정기가 아니라 작가용 상대-추세 advisory**다. 임계·이진 판정·자동 교정 트리거는 전부 의도적으로 넣지 않았다(프로젝트의 G4 측정·가시화 원칙, no-whack-a-mole 원칙, reader_desk '점수 금지' 선례와 정합).

## 1. 'AI 티'의 정체 — 무엇을 고치려는가

### 1.1 두 성분 분해

| 성분 | 정체 | 고치는 자리 | 우리 상태 |
|---|---|---|---|
| **A. 일관성 붕괴** | 설정·호칭·시제·보이스가 회차 간 미끄러짐 | 소스: 장면-스코프 canon 주입(RAG, full-inject 아님), 구조화 상태(NSS류), 결정론 게이트(시제·틱·훅) | 기존 게이트·checker·context-assembly가 담당(별도 라인) |
| **B. 문체 게슈탈트** | 균일한 리듬, 분절적 쉼표, 직유 강박, 종결 단조, 번역투 | 규칙으론 못 고침 → **측정해서 작가에게 추세로 보여줄** 뿐 | 본 문서의 `ai_tell_profile`이 담당 |

성분 A는 "검출 가능한 축"이라 결정론 게이트가 백스톱이고, 성분 B는 "게이트가 못 잡는 게슈탈트"라 **양성 원칙문(8개) + 측정 가시화**로만 접근한다. 이 분리가 `domain/world.py::DEFAULT_STYLE_RULES` 재작성(11개 혼합/부정 규칙 → 8개 양성 원칙)의 근거다.

### 1.2 왜 규칙 열거는 실패하는가 (풍선효과, 실측 확인)

프롬프트에 "쉼표를 줄여라 / 직유를 쓰지 마라" 같은 부정 규칙을 넣으면:
- 모델은 금지된 축을 누르는 대신 **다른 축으로 표현 압력을 옮긴다**(직유 금지 → 은유·체언 비유 증가).
- 규칙이 늘수록 생성이 경직되고 본문 품질이 동반 하락한다.
- 검출 점수를 '최적화 타깃'으로 삼으면 패턴 회피만 학습하는 **군비경쟁**이 된다(RAID 벤치마크가 적대편집만으로 검출 성능 붕괴를 입증).

→ 결론: **검출 패턴은 '교정 규칙'이 아니라 '측정 피처'로만 격하**해서 쓴다. 이것이 비-두더지잡기 노선의 핵심.

## 2. AI 검출기 지형과 신뢰도 평결

### 2.1 검출기를 '판정기'로 쓰면 안 되는 5가지 근거

1. **거짓양성이 치명적** — OpenAI 자체 분류기는 정확도 26%·사람글 오탐 9%로 2023-07 폐기. Stanford HAI: 7개 검출기가 비원어민 영어 에세이를 평균 61%(최악 97%) 오탐, 원어민엔 <10%. **우리 한국어 창작체(파편문·구어체·정형 훅)는 '비표준'으로 분류돼 동형 오탐 위험.**
2. **비영어/CJK 구조적 편향** — perplexity·곡률·cross-perplexity 계열(Fast-DetectGPT, Binoculars, DetectGPT)은 한국어 능통 참조모델 없이는 신호 자체가 왜곡. Binoculars 논문도 저자원 언어 '재현율 붕괴'를 명시. KatFishNet은 log-likelihood/entropy 베이스라인이 한국어에서 AUROC 49~61%(무용)임을 실측.
3. **우회 용이 = 두더지잡기 함정** — RAID: 적대편집·인간글 혼합만으로 검출 성능 급락. '점수 = 진실' 금물.
4. **저-burstiness 함정** — 웹소설 문체는 클리셰·정형 훅으로 본질적 저-burstiness. **인간 작가 회차도 'AI스럽다'고 나올 수 있다.**
5. **LLM-judge 'AI티 0~10'은 비결정·비용·재현불가** — reader_desk가 이미 '점수 금지' 원칙을 세운 이유.

### 2.2 OSS/스킬 지형 (조사 결과)

| 리소스 | 한국어 | 우리 활용 등급 | 판단 |
|---|---|---|---|
| **KatFishNet** (Shinwoo-Park, ACL 2025, arXiv 2503.00032) | 네이티브 | **자질 차용(최우선)** | 모델·perplexity 0콜로 한국어 AI 티를 분포자질만으로. 우리 철학과 가장 정합. 코드 import 아니라 **자질만 재구현**. |
| **im-not-ai** (epoko77-ai, ★3k, 활발) | 한국어 | 자질 차용 | '검출기'가 아니라 규칙기반 윤문 스킬 = 우리가 이미 기각한 두더지잡기. 단 metrics.py 8지표가 KatFishNet과 **독립적으로 쉼표분포+POS다양성+문장길이분산에 수렴** → 이 3축이 무사전 핵심임을 이중 확인. |
| **xlm-roberta-text-detector** (yaya36095) | 부분 | 보조점수(조건부) | 한국어 입력→{HUMAN,AI} 확률을 로컬·MIT로. 단 학습데이터·정확도·갱신일 미공개. **절대 라벨 금지**, 회차 간 상대 추세로만. 지금은 미채택. |
| Fast-DetectGPT / Binoculars / Ghostbuster / DetectGPT | no(영어) | 참고만 | 한국어 점수화엔 무용. 방법론 영감(burstiness, 함수어 엔트로피+문장길이 분산+구두점 분포)만 한국어 등가물로 재구현. |

**KatFishNet의 명시 자질**(논문/리포지토리 기준): 쉼표 분포(에세이 도메인 최강 변별자질), POS n-gram 다양성, 명사 편중(LLM이 명사↑·사람이 종결어↑), 띄어쓰기 준수율, Zipf/Heap 어휘다양성.

**핵심 교차검증**: 이 중 **쉼표 분포**와 **어휘/구문 다양성**이 im-not-ai의 metrics.py 지표와 **독립적으로 수렴**한다 — 이 축이 "한국어 AI 티의 무사전 핵심 신호"라는 가장 강한 증거다. 단 정확히 하면:
- **문장길이 분산/CV**는 KatFishNet의 *명시 축이 아니라* im-not-ai metrics.py + 일반 burstiness 계열에서 온 **보완 신호**다(우리는 척도-불변이라 채택).
- **띄어쓰기 준수율**은 KatFishNet 축이지만 띄어쓰기 정답 레퍼런스가 필요해 **의존성 0 원칙상 미구현**(5장 로드맵 참조).
- **POS 다양성/명사비**도 형태소분석기 의존이라 현재 미구현 — 우리는 의존성 0으로 잡히는 축(쉼표·어휘·종결형 근사)만 가져왔다.

## 3. 우리가 적용한 측정: `ai_tell_profile()`

### 3.1 설계 원칙

- **외부 코드 import 0** — KatFishNet의 *자질 정의*만 재구현(방법은 논문 공개라 안전). 의존성·형태소분석기 0.
- **LLM 0콜 · 결정론** — 회차마다 비용·재현성 부담 없이 항상 계산(reader_desk는 +1콜이라 토글이지만 이건 무료라 always-on).
- **판정 아님 · advisory만** — 절대 임계·이진 판정·자동 재작성 트리거 전부 금지. 작가가 추세를 보고 판단.
- **밀도/CV/이동평균만** — 모든 신호를 길이-불변 형태로(아래 3.3 검증 교훈).

### 3.2 5개 신호 (`engine/quality_gates.py::ai_tell_profile`)

| 키 | 자질 | 방향(AI 의심) | 길이-불변 처리 |
|---|---|---|---|
| `comma_per_100`, `comma_per_sent` | 쉼표 분포(KatFishNet 최강 자질 — 리서치 기준 에세이 도메인 AUROC ~97.6) | 높을수록(영어 학습 흔적의 분절적 쉼표 과다) | 밀도/문장당 — 본래 불변 |
| `sent_len_cv` | 문장 길이 변동계수 std/mean | **낮을수록**(균일 = AI, 사람은 장단 변주). ※액션/대사 회차는 인간도 균일 → 단독 판정 금물 | CV는 척도-불변 |
| `lexical_mattr` | 이동평균 TTR(고유어 roster 제외, window 60단어) | 낮을수록(표현 반복) | **MATTR**(3.3 참조) |
| `ending_diversity` | 종결형 다양성 — 말미 음절 근사(어간 혼입 줄이려 마지막 1음절, 이동평균 window 40) | 낮을수록('~다' 단조). ※형태소분석기 없는 거친 근사 | 이동평균 |
| `simile_per_1k` | 비교 형태소(처럼/마치/듯이/듯한) 밀도 | 높을수록(비유 강박) | 밀도. **닫힌 형태소 클래스의 count 신호**(교정 규칙 아님 → 두더지잡기 아님) |

> `simile_per_1k`이 사전 아니냐는 반론에 대한 답: 처럼/마치/듯이/듯한은 **닫힌 문법 클래스(비교 형태소)**다. 쉼표(닫힌 구두점)를 세는 것과 같은 등급의 형태론 신호이며, "이 단어를 쓰지 마라"는 교정 규칙이 아니라 **카운트만 하는 측정 피처**다. 열린 패턴 사전(번역투 표현 목록 등)을 교정에 쓰는 것과는 다르다.

### 3.3 검증이 드러낸 결함과 수정 (MATTR)

실제 ch10(구 스타일)·ch11(신 스타일)에 돌렸더니:

```
[수정 전 — 원시 TTR]
ch10 len=4277: lexical_ttr=0.699  ending_diversity=0.613
ch11 len=5003: lexical_ttr=0.615  ending_diversity=0.554   ← '신 스타일이 더 AI스럽다'?!
```

이 "퇴보"는 **거짓 신호**였다. 원시 TTR은 텍스트가 길수록 기계적으로 하락하는 **길이 교란(length-confound)**이 있고, ch11이 단지 더 길었을 뿐이다. "회차 간 상대 추세"가 이 측정의 핵심 용도인데, 길이가 다르면 raw TTR 비교가 왜곡된다 → 측정으로서 실격.

수정: 표준적 길이-불변 형태인 **MATTR(이동평균 TTR, 고정 창)**로 교체. 종결 다양성도 동일 창 처리.

```
[수정 후 — MATTR/이동평균 + 종결형 말미음절 추출]
ch10 len=4277: lexical_mattr=0.970  ending_diversity=0.387
ch11 len=5003: lexical_mattr=0.952  ending_diversity=0.402   ← 길이 교란 제거, 거의 평탄
```

길이 교란이 사라지자 거짓 퇴보가 소멸했다. **살아남은 정직한 신호**: `simile_per_1k` ch11(2.94) > ch10(2.56) — 신 스타일이 직유를 더 쓴다(풍선효과 실측), `comma_per_100` ch11(2.86) < ch10(3.07) · `sent_len_cv` ch11(0.511) > ch10(0.489) — 둘 다 신 스타일이 옳은 방향(덜 분절적·덜 균일). 측정-주도가 실제로 검증을 통해 설계를 교정한 사례다.

### 3.4 적용 위치 (wiring)

- `engine/quality_gates.py` — `ai_tell_profile()` 신규 + `chapter_quality_report()`에 `"ai_tell"` 필드 합류.
- `domain/types.py::ChapterRecord` — `ai_tell: dict` 필드 추가(reader_feedback와 동일 advisory 등급).
- `services/copilot.py` — 회차 생성 직후 결정론 계산해 `record.ai_tell`에 저장(reader_desk advisory 블록 바로 뒤, try/except로 측정 버그가 생성을 깨지 않게). **차단·임계·재작성 0.**

### 3.5 의도적으로 하지 **않은** 것

- ❌ 절대 임계/이진 'AI/사람' 라벨 — 거짓양성 위험(2.1), 점수=진실 금물.
- ❌ 검출 점수를 생성 입력의 negative rule로 주입 — 풍선효과(1.2).
- ❌ 점수 미달 시 자동 재작성 트리거 — 군비경쟁(RAID).
- ❌ xlm-roberta 외부 점수 연결 — 검증·재캘리브레이션 전엔 위험. 한국어 분포자질이 자리잡은 뒤 '회차 간 상대 추세' 보조신호로만 선택 검토.
- ❌ 열린 패턴 사전(번역투 목록 등)을 교정 규칙으로 등록 — 유지보수 지옥·두더지잡기.

### 3.6 적대 리뷰가 잡은 추가 교정 (code-critic 루프)

적용 후 적대 리뷰를 한 번 더 돌려 측정-타당성 결함 4건을 잡고 교정했다(검증 리뷰 루프):

| 결함 | 문제 | 교정 |
|---|---|---|
| **종결형 추출 방향 오류** | `s[-3:]`(끝 3글자)는 어간 꼬리(었다/췄다/갔다)를 세서 종결이 단조여도 다양도가 높게 나옴 — **신호 방향이 반대** | 말미 구두점·따옴표 제거 후 **마지막 1음절**만(평서문이 '다'로 수렴 → 단조 AI가 낮은 다양도로 정직하게 잡힘) |
| **roster 미전달** | production 와이어링이 인명/고유어 roster를 안 넘겨 `lexical_mattr` 오염 | copilot.py에서 ontology 엔티티명 + bible 키워드로 roster 구성해 전달 |
| **문서 KatFishNet 오인용** | '문장길이 분산'을 KatFishNet 명시 축으로 귀속, 띄어쓰기 준수율 누락 | 2.2 자질 목록 정확화 — CV는 im-not-ai/burstiness 보완 신호, 띄어쓰기·POS는 의존성 사유로 미구현임을 명시 |
| **sent_len_cv 장면유형 교란** | 액션/대사 회차의 의도적 균일 단문을 'AI'로 오판 가능 | docstring·표에 교란 경고 명시 — 단독 판정 금지, 추세로만 |

→ 측정을 추가하는 작업조차 적대 검증을 거쳐야 '거짓 신호'를 안 내보낸다는 원칙의 실례. blocker는 없었다(차단 경로가 애초에 없음 — advisory 격리가 옳았다).

---

## 4. 소비 모델 (어떻게 읽어야 정직한가)

1. **절대값 금지** — 모든 신호는 **작품 자기 코퍼스의 분위수 대비 상대 추세**로만 해석. ch12가 작품 평균보다 쉼표 z-score가 튀면 "검토해보라"는 신호일 뿐, "AI다"가 아니다.
2. **단일 회차 스파이크 금지** — 이동평균/아크 단위 추세로(G3 시계열 회고와 합류). 한 회차의 한 숫자로 판단하지 않는다.
3. **작가가 최종 판정자** — 측정은 주의를 끌 뿐, 고칠지·무시할지는 작가가 결정. reader_desk·생성 디버그 탭과 같은 advisory 등급.

## 5. 로드맵 (측정이 자리잡은 뒤)

- **회차별 추세 가시화 — ✅ 출하**(연재관리 > 연재현황 '문체 지문 추세'). 작품 자기 **중앙값 대비 상대 추세**(스파크라인 min-max 정규화로 저변동 지표도 보이게) + raw 최근/중앙값 숫자. **판정 누수 차단**: 적대 리뷰 후 '중앙값 대비 +N%' delta 제거(암묵 임계→단일회차 과독 유발), 경고색·임계·교정버튼 0, "AI 판정 아님" 카피. 결측 키는 0으로 위장하지 않고 메트릭별 실측만(median 오염 방지). **stale 소스 차단**: 퇴고 accept/undo가 본문 교체 시 `_recompute_ai_tell`로 재계산(생성·accept·undo 3경로 공유). 미측정(도입 전) 회차는 백필 안내.
- **분위수/z-score 정교화(후속)** — 현재는 중앙값 상대. 코퍼스가 더 쌓이면 분위수 띠·최근 N화 윈도우.
- **POS·띄어쓰기 축 보강(선택)** — 형태소분석기/띄어쓰기 레퍼런스 의존을 감수할 수 있으면 KatFishNet의 POS n-gram 다양성·명사비·띄어쓰기 준수율을 추가(현재는 의존성 0 유지를 위해 제외).
- **xlm-roberta 보조신호(조건부)** — 우리 회차로 자체 검증/재캘리 후에만, 절대 라벨 아닌 상대 추세로.
- **소스측 강화(성분 A)** — 장면-스코프 canon 주입·구조화 상태(NSS)는 별도 라인에서 일관성 붕괴를 줄인다(이 문서의 측정과 보완 관계).
