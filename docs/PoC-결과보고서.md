# AI 웹소설 에이전틱 코파일럿 — PoC 결과 보고서

> 대상: PRD v3(`PRD.md`)의 "기계가 쓰고 작가가 보조하는 회차 단위 RAG 코파일럿" 핵심 기술 가설 검증.
> 백엔드: OpenAI gpt-4.1 + text-embedding-3-small (아키텍처 검증용 — 프로덕션은 Claude/BGE-M3로 교체 가능).
> 코드·산출물: `../poc/` (버전별 보존). 본 보고서는 전 과정의 종합본.

---

## 0. 한 줄 요약 (TL;DR)

- **일관성 엔진은 동작하고 증명됐다** — v2 엔진 11개 게이트(C0~C10) 3회 연속 PASS. 검증기가 실제 생성경로에서 설정 위반을 막고(C9/C10), 순수 결정론 레이어(C0/C7, LLM 0콜)도 실재한다.
- **형식도 웹소설을 달성했다** — 작법 적용 후 회차 평균 5,180자, 웹소설식 줄바꿈·대사·절단 훅, 전 20화 캐논 일관성 유지.
- **그러나 "재미"는 미해결이다** — 독립 편집자 평가 **2.8/10 "유료전환 불가"**. 일관성 엔진은 *"설정이 안 어긋남"(필요조건)*만 보장하지 *"재미있음"(충분조건)*은 보장하지 않는다. 재미는 골격·페이싱·캐릭터·떡밥회수라는 **별도 레이어**다.
- **결론: 토대는 v2로 확정.** v1은 린 레퍼런스로 보존. 다음 과제는 엔진 고도화(동적 온톨로지·흐름) + 재미 레이어.

## 1. 검증 대상 (가설)

회차 N = f( RAG(이전 회차 1..N-1), 온톨로지 SSOT, 골격/비트, 작가지시 )의 **점진 생성 루프** + **비대칭 일관성 아키텍처**:
- 정형 사실(생사·소속·눈색·등급)은 **온톨로지 결정론 lookup으로 "박고"**,
- 서사 배경은 **RAG로 "찾고"**,
- 에이전트 루프(plan→draft→check→reflect)로 생성하며,
- **결정론 검증기**가 위반을 잡고, 자가교정이 수정하고, **비수렴 시 escalation**(작가에게 올림, 침묵 확정 금지).

## 2. 두 버전

| | **v1-linear** (첫 PoC, 보존) | **v2-engine** (재작성, 확정 토대) |
|---|---|---|
| 하네스 | 회차 통짜 draft→check→rewrite (선형) | 타입드 상태머신 + scene 미니루프(국소 재계획) |
| 상태/계약 | dict 흘려보냄 | **Pydantic 타입** — ContextBoard 3슬롯(ground_truth/narrative/authority) 분리 = 비대칭 주입 타입강제 |
| 검증기 룰 | 모놀리식 if + substring(규칙 2개째 깨짐) | **RuleSpec 룰테이블(룰=데이터)** |
| 기억 | 청크-RAG | 청크-RAG + **LLM Wiki**(컴파운딩 페이지·결정론 lint) |
| 색인 | 비멱등(재색인 중복 버그) | **멱등** |
| 관측 | 없음 | obs 훅 → `events.jsonl` |
| 위치 | `poc/v1-linear/` | `poc/v2-engine/` |

## 3. 검증 게이트 결과 (C0~C10)

신뢰등급 분리: **결정론/준결정론은 게이트(binding)**, 의미층·OOV·경계 재현성은 **비구속 보고**(LLM-judge는 게이트 입력 금지).

| 게이트 | 무엇을 증명 | 종류 | v2 | v1 |
|---|---|---|---|---|
| C0 | SSOT 내부 모순 검출 (**LLM 0콜**) | 순수 결정론 | ✅ | ✅ |
| C1 | 룰엔진 위반 recall, 오탐 0 | 준결정론 | ✅ | ✅ |
| C2 | hold-out 일반화 (beat 밖 사망자 부활 본문스캔 탐지·회상 정밀도) | 준결정론 | ✅ | ✅ |
| C3 | LLM 추출 판정 안정성 (n=10) — *코드 결정론 아님(그건 C0/C7)* | 보고 | ✅ | ✅ |
| C4 | 파이프라인 무결성 — 하드 위반 침묵 확정 0 | 통합 | ✅ | ✅ |
| C5 | 자가교정 — 주입 위반 → 하드 0 수렴 | 루프 | ✅ | ✅ |
| C6 | RAG 격리 — 엔티티/아이템 회수 + **as_of 시점 필터(미래 누출 차단)** | 검색 | ✅ | ✅ |
| C7 | **위키 lint**(orphan/dangling, **LLM 0콜**) | 순수 결정론 | ✅ | — |
| C8 | **멱등성**(재색인/재ingest → 중복 0) | 안정성 | ✅ | — |
| **C9** | **결함주입: 위반+교정불가 → 실제 경로 탐지·ESCALATED·미색인** | **핵심 증명** | ✅ | — |
| **C10** | **결함주입: 위반+실제교정 → 통합 루프가 탐지·교정·FINALIZED** | **핵심 증명** | ✅ | — |

- **v2: 11게이트 3회 연속 OVERALL PASS** (재현 안정). **v1: 7게이트(C0~C6) PASS.**
- C9/C10이 이 PoC의 가장 중요한 신호: `_draft`에 위반을 주입하고 `_rewrite`를 항등/실제로 바꿔 `generate_chapter` 실제 경로를 시험 → 검증기가 진짜로 막고, escalation 분기가 실제 발화하며(events.jsonl 기록), 통합 교정이 동작함을 증명.

## 4. 적대 검증 이력 (정직성의 핵심)

이 PoC는 "PASS"를 의심하는 적대 검증을 반복해 통과시킨 게 아니라 **깎아냈다.**

1. **1차 (선형 PoC, code-critic)**: "결정론 위장(의미위반을 결정론으로 셈)·조작된 시험(정답지 공유)·재현성 부재" → 신뢰등급 분리·hold-out·재현성 측정으로 닫음.
2. **2차 (재작성본, code-critic)**: "9개 PASS지만 시스템 핵심주장(검증기가 실제 경로에서 막는가·escalation 동작)을 증명하는 게이트가 0개. LLM이 매번 다른 본문을 썼는데 게이트 수치가 안 변함 = 변동에 둔감" → **C9/C10 결함주입 게이트 신설**·obs 영속화로 닫음.
3. **설계 비평 (디자인 패턴)**: LLM Wiki를 "5화 규모엔 cargo-cult, 토대(finalize 팬아웃·DB) 부재"로 **REJECT 경고**. → v2가 토대까지 구현했으나 **wiki ROI는 여전히 미증명**(아래 §7·§9).
4. **편집자 평가 (재미)**: 자동생성 20화 통독 → **2.8/10**(아래 §7).

## 5. v1 ↔ v2 비교 (공정 head-to-head, 동일 골격)

작법 적용 **전**, 동일 20화 골격으로 양쪽 실행한 결과(상세 `../poc/COMPARISON.md`):

| 지표 | v1 | v2 |
|---|---|---|
| 완주 / 하드위반 | 20/20 / **0** | 20/20 / **0** |
| escalation | 0 | 0 (이 골격엔 위반 압박 없음) |
| 위키 | 없음 | 4페이지 + orphan lint |
| 비용 | 62콜 / 60k토큰 | 143콜 / 176k토큰 (**≈2.5배**) |

**핵심 결론**:
- **엔진 본질은 v2 우위** — 타입강제·룰데이터화·멱등·LLM Wiki·C9/C10 증명. outline·작법과 무관한 구조적 우위.
- **단, 클린 골격에선 일관성·완주가 v1과 동률**인데 v2가 ~2.5배 비싸다 → "v2 우위는 무조건이 아니라 **조건부**"(위반多·장거리추적·증명된 안전망 필요시).
- escalation은 **골격 의존** — 다른(어려운) 골격에선 v2가 ch16에서 "죽은 강도현 결말 부활"을 실제 차단했으나, 클린 골격에선 양쪽 0.

## 6. 20화 스케일 + 작법 적용 (형식 해결)

초기 생성물은 회차 ~1,200자·빽빽한 단락으로 **"웹소설 같지 않다"**는 문제가 있었다(작법 사양 부재). 실제 웹소설 관습을 조사(회차 5,000자·웹소설식 줄바꿈·대사 50%+·단문·절단신공)해 **작법 레이어(`WEBNOVEL_STYLE`)를 v2에 적용**:

| | 작법 적용 전 | 적용 후 (v2 20화) |
|---|---|---|
| 회차 분량 | ~1,200자 | **평균 5,180자** (총 103,618자) |
| 포맷 | 빽빽(비웹소설) | 웹소설식 줄바꿈·대사 중심·절단 훅 |
| 일관성 | — | **전 20화 FINALIZED, 하드위반 0, 캐논 준수**(강도현 사망 유지·회상으로만) |
| 위키 | — | 4페이지 컴파운딩 + orphan lint 탐지 |
| 비용 | — | 183콜 / 413k토큰 |

> 형식·일관성·시점안전(과거 회차 누출 차단)·캐논 유지는 **달성**. 단 이것은 "읽을 만함"의 **필요조건**이다.

## 7. 재미 평가 (독립 편집자 에이전트) — 미해결

엔진 저자 편향을 배제하려 베테랑 웹소설 편집자 관점 에이전트에게 v2 20화 전편 통독 평가를 받음:

- **종합 2.8/10, "무료 5화 안에 연독률 5% 밑, 유료전환 0 수렴 — 런칭 불가"**
- 실패 지점(구체):
  - **페이싱 붕괴(가장 치명)**: 실질 사건 3개를 13화로 늘려 "단검 조사·감시자 의심·강도현 환청" 반복.
  - **캐릭터 평면·목소리 미분화**: 둘 다 "같이 가자/지켜줄게" 3종 세트, 화자 구분 불가.
  - **사이다 부재·떡밥 미회수**: 주인공이 시종 당하고 자책. 단검 정체·팀장 배신 떡밥 미회수, 새 떡밥만 증식.
  - **훅 반복**("문 손잡이가 돌아간다" 5번+), **완결 실패**(20화가 1화 재탕 에필로그 + 새 떡밥).
  - **🐞 생성 결함**: 13화에 영어 `oozed out`이 한국어 본문에 노출.
- **근본 원인**: 작법은 "어떻게 쓸지"만 고쳤다. 재미는 **"무엇을 어떻게 전개하느냐"**(골격 사건밀도·페이싱·캐릭터·떡밥회수)라는 별도 레이어이며, 일관성 엔진은 이를 보장하지 않는다.

## 8. 결론

| 층위 | 상태 |
|---|---|
| 일관성 엔진 (검증·escalation·온톨로지·RAG·wiki) | **검증됨** (v2 11게이트 3회 PASS, C9/C10 핵심주장 증명) |
| 형식 (분량·줄바꿈·문체) | **해결됨** (v2 평균 5,180자 웹소설 포맷) |
| 재미 (페이싱·캐릭터·떡밥·사이다) | **미해결** (2.8/10, 별도 레이어) |

→ **토대 = v2-engine 확정.** v1-linear는 보존 레퍼런스. **일관성 엔진은 "안 어긋남"이라는 필요조건을 푼 것이지, "읽을 만한 웹소설"이라는 충분조건을 푼 것이 아니다.**

## 9. 정직한 한계 (M2+ 과제)

- **의미층(재각성 등)은 LLM 판단** — 우회 표현("두 번째 권능의 문") 탐지 실패. 자동확정 금지·escalation으로만.
- **OOV escalation 부분(~50%)** — 추출기가 속성 주장 자체를 인식 못 하면 침묵 통과.
- **본문↔본문 사실 모순**(3화 '단검' vs 후속 '독')은 SSOT에 없으면 사각지대.
- **wiki ROI 미증명** — 컴파운딩·orphan lint는 동작하나, 청크-RAG 대비 생성 연속성 개선은 A/B 미격리. orphan은 payoff-edge 자동추출 미구현으로 **false-positive 가능**.
- **재현성은 결정론이 아님** — C3는 LLM 추출 안정성 측정. 순수 결정론은 C0/C7(LLM 0콜)뿐.
- **미검증**: 규모 검색 품질(청크 수천 p@k)·장거리 드리프트 누적·비용/지연·작가 실시간 동시편집. 단일 장르·인물 3·규칙 1.
- **작법 비대칭**: 현재 `WEBNOVEL_STYLE`는 v2에만 적용(v1 미적용) — 두 버전 출력 품질 격차의 상당부분은 엔진이 아니라 이 적용 차이.

## 10. 다음 과제 (식별됨)

### 10.1 엔진 고도화 (논의됨, 미착수)
- **동적 온톨로지**: 회차 진행에 따라 새 인물·관계·상태 변화를 캐논에 자동 반영(추가·진행만 허용, 기존 캐논 모순 시 escalation). 현재 온톨로지는 시작 시 고정.
- **글의 흐름**: 회차마다 독립 프롬프트라 흐름이 끊김. 직전 회차 raw 전문 + 회차 내 직전 장면 전문 + "이어가기" 지시 주입(장거리 사실은 RAG/캐논 유지, lost-in-middle 회피). 선택: 회차 통짜 1프롬프트 생성.
- **wiki payoff-edge 자동추출**: orphan lint 정확화 + 떡밥 회수 강제.

### 10.2 재미 레이어 (별도)
- 골격 사건밀도 "한 화 한 사이다", 회차가 핵심사건을 반드시 진전(조사·잡담·내면독백 반복 차단).
- 붉은 눈 능력(진실/감정 간파)을 심문·배신자 색출에 적극 활용 = 차별화 + 사이다.
- 캐릭터 보이스 분리, 떡밥 회수 강제, 절단 훅 다양화, 영어 누출 필터.

### 10.3 M2 측정
- wiki 주입 ROI A/B, 규모 검색 p@k, 어려운 골격에서 escalation/일관성 우위 실측 → v2의 ~2.5배 비용 정당화.
